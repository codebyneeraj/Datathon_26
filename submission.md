Submission Guidelines

Please ensure that your team submits the following details as part of the final solution submission:

    Prototype Brief
    Share a concise overview of your solution, including:
        Problem statement addressed
        Key features and functionalities
        Technology stack used
        Proposed impact and use case
    Public GitHub Repository Link
    Submit a publicly accessible GitHub repository containing:
        Complete source code
        Proper README documentation
        Setup and execution instructions
    Demo Video Link
    Share a publicly accessible demo video link via:
    The demo video should clearly showcase:
        Google Drive (public access enabled), or
        Unlisted/accessible YouTube link
        Problem statement overview
        Working prototype demonstration
        Key functionalities and workflows
    Deployed Solution Link
    Submit the live deployment link of your solution.
    Important: Deployment must be done exclusively on the Catalyst platform to qualify for evaluations.
    Submission Template Compliance
    Teams must use the official submission template provided by the organizers to submit the complete solution. Submissions in any other format may not be considered for evaluation.

    Submission Completeness
    Ensure all links are working and accessible before final submission. Incomplete or inaccessible submissions may lead to disqualification from the evaluation process.
